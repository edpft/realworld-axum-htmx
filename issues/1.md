## Context
Currently, the system does not verify email ownership during registration, which is a security best practice. Users receive their auth token immediately after registration per the RealWorld spec.

## Why Later
- Out of scope for RealWorld spec v1
- Adds complexity to the registration flow
- Requires async email sending capability

## What Would Be Needed
- EmailService port for sending verification emails
- EmailVerification aggregate to track verification state
- Verification token generation and expiration
- Updated registration flow to require email confirmation

## Security Consideration
OWASP recommends email verification to prevent:
- Account takeover (registering with someone else's email)
- Spam registration
